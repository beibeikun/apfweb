# 流程编排

> [!IMPORTANT] 版本时效性声明
> 当前文档对应版本：`v0.7.0`
> 
> 由于项目正处于**早期高频迭代阶段**，功能变更较为频繁，文档更新速度可能暂时无法完全同步。如发现内容缺失，请等待文档更新或联系开发者。

> [!WARNING] 配置声明
> 本模块的部分/全部功能依赖于 **大模型（LLM）连接**。请确保在使用前已在 [系统设置](/settings) 中完成模型连接设置。

## 1. 简介
流程编排模块为用户提供了一个直观的可视化工作区。通过简单的拖拽节点和连线，用户可以将不同的文件处理步骤（如排序、重命名、压缩、归档等）自由组合成一条全自动的工作流水线。该模块支持简单的单线流程，也支持复杂的多分支和多线汇合的网状流程。

## 2. 界面概览与画布操作

在流程编排界面，用户可以直接在中央的画布上进行可视化配置。支持以下快捷操作：

* **添加步骤**：点击左侧面板中的可用步骤按钮，即可将其添加到画布中央。
* **连接步骤**：使用鼠标左键按住节点右侧的“输出端口”，拖拽并连接到下一个节点左侧的“输入端口”。
* **缩放画布**：在画布区域滚动**鼠标滚轮**即可放大或缩小视图（支持触控板双指缩放）。
* **平移画布**：在画布空白处**双击并按住鼠标左键**即可拖动视角（也可使用鼠标中键，或按住 `Alt` 键 + 鼠标左键拖动）。
* **删除节点或连线**：使用鼠标点击选中需要删除的节点或连线（可框选多个），按键盘上的 `Delete` 或 `Backspace` 键即可删除。

## 3. 可用节点类型

左侧面板提供了多种处理节点，按功能分为三大类：

1.  **起点节点（仅有输出端口）**
    * **开始**：**每个工作流必须包含且仅包含一个“开始”节点**，它是流水线执行的唯一源头。
    * **提供参数**：用于在流程最前端统一预设各项配置（如输出路径、命名规则等），它只能作为辅助起点，其参数会自动向下游传递。
2.  **处理节点（既有输入也有输出端口）**
    * 支持连成链条或进行分支。包含：**整理排序**、**改名压缩**、**图片压缩**、**提取主图**、**更换格式**、**图片校对**、**图片检查**、**输入参数**。
3.  **终点节点（仅有输入端口）**
    * 位于流水线的末端，处理完后不可再连接其他下游节点。包含：**备份归档**、**目录保存**。

## 4. 详细使用方法

### 步骤一：搭建基础工作流
1. 点击左侧的 **“开始”** 按钮，在画布上放置唯一的源头节点。
2. 根据用户的需求，依次点击左侧的其他功能节点（如“整理排序”、“图片压缩”等）。
3. 将上一步的输出端口，用鼠标拖线连接到下一步的输入端口。

### 步骤二：配置节点参数
1. 在画布中单击选中某一个节点。
2. 界面右侧会显示该节点的 **“步骤参数 (JSON)”** 面板。
3. 点击右侧面板上方的 **“编辑参数”** 按钮，系统会弹出可视化的参数配置窗口。
4. 在弹出的窗口中，通过直观的表单填写该步骤所需的设置（如路径、规则等），填写完毕后确认保存。

### 步骤三：保存与加载编排方案
* **保存编排**：点击界面下方的 **“保存编排…”** 按钮，可以将当前画好的流程图和所有参数保存为一个 `.json` 文件，方便日后重复使用。
* **加载编排**：点击 **“加载编排…”** 按钮，可以选择之前保存的 `.json` 工作流文件，将其一键恢复到画布中。

### 步骤四：运行工作流与查看日志
1. 确认所有节点连接正确，且每个步骤的参数均已配置完毕。
2. 点击界面下方的 **“运行工作流”** 按钮，系统将开始全自动处理。
3. 在运行过程中，界面最下方的 **“运行日志”** 区域会实时滚动显示当前正在执行的步骤和处理信息。运行完毕后，系统会弹出完成提示并播放提示音。

## 5. 常见问题与解决办法

在配置与运行工作流时，用户可能会遇到以下情况。请对照表格进行快速排查：

| 问题现象 | 可能原因 | 解决办法 |
| :--- | :--- | :--- |
| **点击“运行工作流”无反应或提示无法启动** | 画布中缺少“开始”节点，或者错误地放置了多个“开始”节点，导致系统无法确定执行的唯一源头。 | 请检查画布，确保整条流水线**有且仅有一个“开始”节点**。如果需要预设多组参数，请添加“提供参数”节点，而不是添加多个“开始”节点。 |
| **多条线路汇合到同一步骤时运行失败** | 两条或多条上游线路处理的文件不在同一个文件夹内（文件夹上下文不一致），系统无法将它们合并处理。 | 梳理连线走向，确保交汇到同一个节点的所有上游步骤，其**最终输出的文件夹必须是相同的**。若文件夹不同，请将其断开，分别连接到独立的处理节点。 |
| **处理结果错误，或提示缺少路径/命名规则** | 当前步骤的某些必要参数留空，并且上游节点（如“提供参数”节点）也没有预设相关的值，导致该步骤缺少执行指令。 | 选中报错的节点，点击右侧的“编辑参数”补全相关信息；或在流程最前端添加“提供参数”节点，统一设置全局通用的规则（下游会自动补全）。 |
| **右侧的“编辑参数”按钮是灰色的** | 当前画布中没有选中任何具体的步骤节点，系统无法确定用户要编辑哪一步。 | 用鼠标在画布中**单击选中目标节点**（选中后节点会出现高亮边框），此时右侧的“编辑参数”按钮即可恢复点击。 |
| **加载编排方案时提示“无效的工作流文件”** | 选中的文件已损坏，或者并非由本软件“保存编排”功能导出的专属 `.json` 文件。 | 请重新选择之前正确保存的 `.json` 格式工作流配置文件。 |
| **运行报错或日志提示执行“失败”** | 节点的参数填写有误（如输入了不存在的文件夹路径），或者当前文件的状态不符合该步骤的要求。 | 查看界面底部的“运行日志”了解具体的错误细节，然后选中对应的节点点击“编辑参数”，检查相关路径或规则是否填写正确。 |
| **点击运行时提示“已有任务正在运行”** | 上一次启动的工作流任务正在后台处理中，还未完全结束。 | 请观察底部的运行日志，耐心等待当前流程全部执行完毕（完成后会播放提示音），然后再启动新的任务。 |
