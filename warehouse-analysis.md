# 仓库分析

## 作用简述

对已配置的 **图片仓库** 做扫描统计：合同单数、拍品 zip 数、超过一年的拍品数量、仓库总体积、老旧拍品体积合计，以及不符合约定的 **异常项** 明细。结果可 **导出 Excel**；上次成功的分析会缓存在本机，启动时可 **自动加载**（带时间说明）。仅勾选参与扫描的仓库会更新对应缓存条目，其余曾在本机保存过的仓库仍保留上次数据。

## 统计规则（与界面说明一致）

- **合同单**：仓库根下除 `thumbnail` 外的一级子文件夹数量。
- **拍品**：各合同目录下名为 `拍品编号.zip` 的文件，须为有效 ZIP；主文件名须为 ASCII 字母、数字与连字符，不得含 `_副本` 等副本后缀或尾部 ` (1)` 类重复标记。
- **超过一年**：以 zip 修改时间是否早于当前 365 天为准。
- **仓库总大小**：根目录下全部文件递归合计（不跟随符号链接）。
- **超过一年拍品总大小**：仅统计上述有效拍品 zip 中超过一年的文件体积。
- **异常项**：非 `.zip`、命名不符、伪 zip、合同目录内不应存在的子文件夹等。

## 操作步骤

1. 在可选仓库列表中勾选要分析的一个或多个仓库（列表来自 [系统设置](/settings)）。可用「全选 / 全不选」。
2. 点击「开始分析」；完成后表格展示各仓库指标与合计行（含未勾选但仍有缓存的仓库行）。
3. 若有异常，下方「异常拍品 / 文件明细」表列出仓库、原因与相对路径。
4. 导出分析结果：将当前结果导出为 Excel，保存到默认输出根路径（需事先在系统设置中配置且目录有效）。
5. 清除所选仓库的缓存：勾选要处理的仓库后点击，仅从本地缓存中移除这些仓库的分析条目，其他仓库的缓存保留。
6. 清除上次结果：删除整份本地缓存快照（界面不再显示任何上次结果）。

## 常见问题

- 至少勾选一个仓库再分析。
- 导出前请配置有效的默认输出根路径。
- 启动时若看到「上次于 … 保存的分析结果」，表示为缓存数据；勾选对应仓库后点击「开始分析」可以仅更新这些仓库的条目。
- 若清除所选仓库的缓存提示与缓存不一致，请确认设置中的仓库名称与路径是否与保存缓存时相同。
